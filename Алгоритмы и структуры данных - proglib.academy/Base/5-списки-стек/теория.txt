Асимптотика операций в списке
    поиск (как у поиска в массиве)
        в лучшем случае: T(n) = O(1)
        в худшем случае: T(n) = O(n)

        доп. память M(n) = O(1)
    
    добавление, удаление (время работы константное)
        T(n) = O(1)
        M(n) = O(1)
    
    объединение двух односвязных списков (зависит от длины первого)
         T(n1, n2) = O(n1)
         доп. память M(n1, n2) = O(1)

Сравнение списков с массивом
    
    Недостатки списков:
        нет доступа по индексу (тут сомнительно, скорее актуально для С/С++)
        расходуется доп. память
        узлы могут располагаться в памяти разряжено
    
    Преимущества списков:
        быстрая вставка узла
        быстрое удаление узла


Асимптотика стека на базе динамического массива
    push:
        T(n) = O(1) - амортизированная оценка. Совпадает с временем push_back
    pop:
         T(n) = O(1) - всегда
